Et enkelt spill

Et klikk oppdaterer spillet

En viktig del av et spill er hyppige oppdateringer for at spillet som programmeres skal kunne ha funksjoner som en stoppeklokke som viser tiden til spilleren konstant eller kanskje skal noe animeres i spillet.

Tidligere i kapittel 5 i Kode 2 boken forklartes det at man kan legge det du ønsker skal repeteres i en funksjon og så kalde funksjonen igjen og igjen. Denne gjentagelsen kan gjøres ved en onclick eller mouseover strategi, eller automatisk. Vi skal se mer på den automatiske senere i kapittelet, men først skal vi se på et program der poengene til spilleren går opp med to hver gang de klikker på musen

Variabelen poeng er en global variabel for at den kunne bygges på å ikke bare defineres som 0 hver runde.

Endrer plasseringen av et img-element

Nå videre kan vi ta programmet til et nytt nivå ved å legge til en funksjon som flytter musen til et tilfeldig sted innenfor ruta. Altså skal vi bytte y og x koordinatene til img-elementet hver gang spilleren klikker på den. Vi vet at en skjerm har et koordinatsystem der koordinatene er (0,0) oppe i venstre hjørne. Som style skriver vi inn bredden på figuren og hvor lang tid den skal bruke på overgangen mellom to posisjoner.

Nå lager vi en funksjon for å finne x og y posisjonen til img-elementet. For å finne passene kordinater tar vi et tilfeldig tall ved å bruke Math.random() ganget med skjermbredden minus figurbredden for at den ikke skal komme utafor. Vi finner kordinatene for top og left og skriver det slik: mus.style.left = xpos + «px»

  position: absolute;
  transition: 0.2s;
  }
</style>

Game loop - spillet oppdateres kontinuerlig

En game loop -også kalt en rekursiv funksjon- er en funksjon som starter av seg selv og kaller på seg selv når de har gjort det de skal. For vår mus kode så kan vi legge til et tall som øker med 5 hver gang funksjonen er kalt. Når vi lager rekursive funksjoner er det greit å vite at de må ha en betingelse som avslutter de for å ikke få feilmelding. Derfor legger vi inn e betingelse på funksjonen. Så lenge tallet er mindre enn 100 så løper den løpsk.

<span id="visTall"></span>
<script>
 let visTall = document.getElementById("visTall")
 let tall = 0
 function oppdatertTall() {
   tall = tall + 5
   visTall.innerHTML = tallet
   oppdaterTall()
 }
 oppdaterTall()
</script>

Forsinkelse i Game loopen

Ofte er det ikke bra at datamaskinen går så fort som den gjør. For eksempel hvis du vil lage en slags animasjon eller andre endringer vi gjerne vil se imens den går. Da lager vi forsinkelser. En måte å gjøre dette på er requestAnimationFrame(...) hvor du kan velge hvor mye funksjonen burde forsinkes med. Hvis man ikke skriver (minFunksjon, tid), men heller bare (minFunksjon) så vil den gå vanligvis 60 ganger i sekundet (alle pc-er er forskjellig så dette kan varligere). Vi kan gjøre om forrige eksempel slik:

function tellOpp() {
  divTall.innnerHTML = tall
  tall += 1
  if(tall <= 1000) {
   requestAnimationFrame(tellOpp)
  }
}

Bevegelse i spill med canvas

Animere med HTML-elementer, canvas eller SVG?

Det er flere måter å animere i kode. I løpet av Kode 2 har vi gått igjennom noen muligheter. Man kan bruke vektorgrafikk med SVG eller canvas elementer, eller mer simpelt med å endre posisjon og størrelse på html-elementer

Det er absolutt ingen fasit, med ved akkurat spill-programmering har canvas mye fordeler ved at canvas-elementet består bare av et enkelt bilde som oppdateres hver gang det skjer en forandring. Med animering ved hjelp av html eller SVG må alle bestanddelene i spillet være egne html-elementer. Hvis du ønsker da å animere regn eller snø og må ha flere hundre av dem så vil dette da bli tungt. Ulempen med canvas er at det er vanskeligere å fange opp at bestanddelene treffer hverandre.

Å animere i canvas

Canvas er et html-element, men har litt andre regler når man koder inn egenskapene. Her lager vi et canvas-element med størrelsen 100*400 piksler for å sette inn animasjonen vår her.

Nå oppretter vi en firkant og gir den noen egenskaper i JavaScript. Vi setter farge, størrelse, fart og retning. Fart og setningen setter vi til 0.

Nå setter vi farten per oppdatering. Hvis vi setter xfart til 2 vil den bevege seg 2 piksler, imens hvis den er 0 står den stille. Hvis vi setter xretning til 1 så har den 1 piksel til høyre, imens hvis vi setter den til -1 så går den en til venstre, og null er igjen i hvil. Så får vi bevegelsen ved å gange farten med retningen.

Nå må vi lage en funksjon som starter bevegelsen ved å hente egenskapene ved dot-notasjon.

 </script>
</body>

2. Tegne en ball som flytter seg

Nå skal vi tegne inn ballen så vi lager igjen et objekt som inneholder all informasjonen til ballen som fart, retning, farge og størrelse. Deretter bruker vi dot-notasjon igjen for å hente frem denne dataen i funksjonen tegnBall(). Denne funksjonen lager sirkelen ved hjelp av arc og også forandrer xpos og xpos til neste gang den kjører.

let ball = {
  radius: 7,
  xpos: 100,
  ypos: 100,
  farge: "yellow",
  xretning: -1,
  yretning: 1,
  xfart: 4,
  yfart: 4,
}
function tegnBall() {
  ctx.beginPath()
  ctx.arc(ball.xpos, ball.ypos, ballradius, 0, Math.PI*2)
  ctx.closePath()
  ctx.fillStyle = ball.farge
  ctx.fill()
  ball.xpos = ball.xpos + ball.xfart * ball.xretning
  ball.ypos = ball.ypos + ball.yfart * ball.yretning
}

3. Snu retningen på ballen når den treffer en kant

Nå skal vi snu ballen når den treffer den øverste, den nederste eller den vestre veggen (høyre vegg kommer senere). Vi gjør dette ganske enkelt ved å bytte 1 til -1 eller -1 til 1 når de treffer veggene. De tre posisjonene vi må passe oss for da er når venstre kant av ballen er >= 0, når ballen er >= banehøyde og når øvre kant av ballen er &lg= 0. Husk at på sirkler så vil du gå til ballens radius ikke diameter.

function skjekkOmBallTrefferVegg() {
  if(ball.xpos <= ball.radius) {
   ball.xretning = 1
  }
  if(ball.ypos + ball.radius >= bane.hoyde) {
   ball.yretning = -1
  }
  if(ball.ypos <= ball.radius) {
   ball.yretning = 1
  }
}

4. Tegne en racket og styre den med piltaster

Vi skal nå starte å lage racketen. Vi starter med å definere den som et objekt og legge til en del egenskaper. Deretter trengs det en funksjon som tagner den. Nå skal vi først lage en funksjon på document.onkeydown som sjekker om du har tastet inn opp eller ned knappen på piltastene og ut ifra det gjør om retningen til funksjonen. Deretter er det viktig at vi sørger for at racketen står stille om vi slipper taket på tastene. Dette gjør vi ved å lage en funksjon på document.onkeyup som igjen tittet etter riktig tallkode, men også retningen. Så nå for at racketen skal stoppe ved enden så legger vi til en if-setning som ser om racketen er utafor canvas-elementet. Vi legger denne if-setningen inn i funskjonen tegnRacket().

5. Sjekke om ballen treffer racketen

Nå må vi passe på at racketen faktisk fungerer som en racket. For å finne et punkt der racketen er i kontakt med ballen så må den oppfylle alle disse forutsetningene:

Nå i koden må vi sette inn disse forutsetningene som boolske variabler. I dette trinnet er det viktig å huske på radiusen til ballen. Så lager vi en if-setning som sjekker om alle disse er true og hvis de er det så snus ballens x-retning til -1.

function skjekkOmBallTrefferRacketen() {
  let ballenErTilVenstre = ball.xpos + ball.radius < racket.xpos
  let ballenErTilHoyre = ball.xpos - ball.radus > racket.xpos + racket.bredde
  let ballenErOver = ball.ypos + ballradius < racket.ypos
  let ballenErUnder = ball.ypos - ball.radius > racket.ypos + racket.hoyde
  if(!ballenErTilVenstre && !ballenErTilHoyre && !ballenErOver && !ballenErUnder) {
   ball.xretning = -1
  }
}

6. Å telle poeng og ta vare på reckorden

For å gjøre spillet mer interessant så ønsker vi nå å legge til poeng og rekord. Hver gang ballen treffer racketen får du et mer poeng og hvis du får mer enn rekorden så lagres den nye rekorden i localStorage. 7. Sjekke om ballen er utenfor banen

Nå til slutt må programmet vite når det er GAME OVER. Dette gjør vi ved å sette opp en if-setning som ser om ball.x > bane.bredde + ball.radius*2, hvis det er det så setter den theGameOn til false.

function skjekkOmBallErUtenforBanen() {
  if(ball.x > bane.bredde + ball.radius * 2) {
   theGameIsOn = false
  }
}

En motspiller

Vi kan lage en motstander som aldri taper ved å gi den samme y-posisjon som ballen alltid og få ballen til å sprette tilbake når den treffer denne racketen.

let motspiller = {

function tegnMotspiller() {
